# 频道页桌面端独立滚动设计

## 背景

当前频道页左侧标签栏、频道列表栏与右侧频道详情区虽然各自存在滚动容器，但整体高度仍跟随页面主内容区域，用户在桌面端浏览时会感知为和频道详情区绑定滚动。需求是将频道页限制在桌面端第一视窗内，左侧标签栏、频道列表栏、右侧详情内容区分别独立滚动。

## 目标

- 只处理桌面端。
- 频道页整体高度锁定在首屏可用高度内。
- 标签栏、频道列表栏、频道详情内容区分别独立滚动。
- 频道详情头部保持固定，详情内容区独立滚动。
- 移动端继续保持当前自然流式布局与滚动行为。

## 非目标

- 不调整移动端交互。
- 不重构频道页数据逻辑、路由逻辑或组件结构。
- 不修改标签筛选、频道选择、详情页 tab 逻辑。

## 现状问题

- `main-content` 当前未限制首屏可用高度，频道页容器会跟随内容自然撑开。
- `channel-page-shell` 虽然使用 `height: 100%`，但上层没有稳定的首屏高度约束，导致高度来源不够刚性。
- 左侧标签列表、频道列表、右侧详情内容区的滚动容器都依赖外层高度传递，因此滚动边界并不完全独立。

## 方案

### 1. 桌面端建立固定可用高度

在桌面端为页面主内容区域提供稳定高度约束，使频道页可以使用“首屏剩余高度”而非内容自撑高度。

约束方式：

- `app-layout` 保持整页纵向布局。
- `main-container` 与 `main-content` 在桌面端允许子元素继承可用高度。
- `channel-page-shell` 使用固定可用高度并 `overflow: hidden`，成为频道页内部滚动的边界。

实现上优先使用纯 CSS 与 `100dvh` 计算，不引入 JS 测量逻辑。

### 2. 左侧区域拆分为两个独立滚动区

左侧边栏继续保持现有双栏结构：

- 标签面板：顶部搜索框固定，标签列表滚动。
- 频道面板：顶部频道搜索区固定，底部统计/操作区固定，中间频道列表滚动。

要求：

- 标签列表滚动不影响频道列表滚动。
- 频道列表滚动不影响右侧详情滚动。
- 左侧整栏在固定高度内不再随右侧详情内容增长。

### 3. 右侧详情区拆分为“头部固定 + 内容滚动”

右侧主列延续当前结构：

- 头部信息区固定在主列顶部。
- `channel-content-scroll` 作为唯一滚动容器承载视频列表或数据洞察内容。

要求：

- 切换视频/洞察 tab 时，滚动仍发生在详情内容区内部。
- 视频列表工具栏 `sticky` 行为继续相对于详情内容区生效。

### 4. 移动端保留现状

当前移动端媒体查询已将频道页切为纵向流式布局，并将整体高度改为 `auto`。该行为保留，不做首屏锁定。

## 样式修改范围

主要在以下样式范围内调整：

- `src/index.css`
  - `main-container`
  - `main-content`
  - `channel-page-shell`
  - `channel-main-column`
  - `channel-content-scroll`
  - `sidebar-dual-layout`
  - `sidebar-tag-panel`
  - `sidebar-channel-panel`
  - 相关桌面端/移动端媒体查询

如有必要，可在频道页组件上补充最小 className，但不引入新的状态逻辑。

## 验收标准

桌面端：

- 打开频道页后，页面整体保持在第一视窗内，不因详情内容变长拉伸出整页滚动。
- 标签列表可单独滚动。
- 频道列表可单独滚动。
- 右侧详情内容可单独滚动。
- 频道详情头部在详情滚动时保持固定可见。
- 视频列表工具栏的 sticky 行为保持正常。
- 左右区域互不影响滚动位置。

移动端：

- 频道页保持现有纵向流式布局。
- 不出现因为桌面端改造导致的内容截断或无法滚动。

## 验证方式

- 桌面端手动验证：
  - 标签数量很多时滚动标签栏。
  - 频道数量很多时滚动频道列表。
  - 视频列表很多时滚动详情区。
  - 切到数据洞察 tab 再验证详情区滚动。
- 运行前端构建确认样式与组件编译通过。

## 风险与控制

- 风险：上层容器高度链不完整，导致子滚动容器 `min-height: 0` 失效。
  - 控制：同时检查 `main-container`、`main-content`、`channel-page-shell`、左右列容器的高度链与 `overflow` 设置。
- 风险：移动端继承桌面端固定高度导致截断。
  - 控制：明确将固定高度逻辑放在桌面端范围内，保留现有移动端媒体查询覆盖。
- 风险：视频列表 sticky 工具栏失效。
  - 控制：确保 `channel-content-scroll` 仍是详情唯一滚动上下文。
